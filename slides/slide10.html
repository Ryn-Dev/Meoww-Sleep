Dev Workflow

Istilah Pengembangan

Bagaimana para developer bekerja sama, menyimpan kode, dan mengatasi masalah.

Git

"Mesin waktu" untuk kode. Mencatat setiap perubahan sehingga kita bisa kembali ke versi masa lalu jika ada kesalahan.

Repository (Repo)

Folder proyek digital (brankas) tempat menyimpan seluruh kode, biasanya di hosting seperti GitHub.

Deploy

Proses "menerbitkan" website dari komputer lokal ke server internet agar bisa diakses semua orang.

Bug

Kesalahan atau cacat dalam program yang membuatnya error atau berjalan tidak sesuai harapan.

Debug

Aktivitas detektif: mencari penyebab bug, memperbaikinya, dan memastikan program berjalan lancar kembali.

Development Lifecycle

Fixing Bugs
1. Coding (Local)
2. Git Commit
3. Push to Repo
4. Live / Deploy
